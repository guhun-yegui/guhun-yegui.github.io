# 1. 代码层面

  • 1.改变样式时尽可能合并利用 CSS 去更改，减少修改样式的次数
  • 2.操作 dom 时尽可能批量操作
  • 3.减少回流和重绘
  • 4.尽可能的复用代码，抽取公共 JS 和 CSS
  • 5.事件函数的节流和防抖

# 2.页面加载

  • 1.dns 预解析
  • 2.使用 cdn
  • 3.静态资源的压缩与合并
  • 4.减少 https 请求
  • 5.服务端渲染 ssr
  • 6.图片懒加载

# 2. vue 项目优化

# 1. 代码层面优化

  • 1.v-if 和 v-for 不要在一起使用。v-if 的条件通过函数来处理
  • 2.v-for 中加上 key，对于虚拟 dom 树查找提高性能
  • 3.操作 dom 不要使用 js 原生的方式来操作。用 vue 提供的三种方式来操作 比如，ref、自定义指令 el、事件中的话用 e.target 获取 dom
  • 4.尽量不要在前端进行大量的数据处理
  • 5.合理使用 keep-alive 来缓存页面数据，跳过 created,mounted 钩子，他有自己特定的钩子 activted 等
  • 6.路由懒加载通过 import 配合箭头函数，还有其他的方式 require
  • 7.组件懒加载，异步加载
  • 8.服务端渲染 ssr，优化 seo，与首屏白屏问题。
  • 9.把组件中的 css 提取成单独的文件
  • 10.使用字体图标或者 svg 来代替传统的 Png 等格式的图片
  • 11.vuex 里少塞点东西，多了会卡

# 2. webpack 层面优化

  • 1.去除无用代码 treeShaking.
  • 2.减小 app.js 的体积，提取公共代码
  • 3.通过按需引入第三方库，或者有些资源可以通过 script 标签引入。
  • 4.关闭 SouceMap
  • 5.webpack 对图片进行压缩等处理

# 3. web 层面优化

  • 1.浏览器缓存的使用
  • 2.开启 gzip 压缩
  • 3.CDN 的使用，减少路由转发的次数，就近访问资源
  • 4.使用 chrome 的性能分析工具，查找性能瓶颈
  • 5.dns 预解析
